## コマンドの指示

- `rm -rf` を実行したい場合、代わりに `gomi -rf` を使用してください. (ゴミ箱に移動されます)

## 一般的な指示

- main ブランチにいる場合は該当のブランチに切り替えるか、新たな作業ブランチを作成してください。
    - 作業ブランチにいる場合は、こまめに commit を行ってください
    - 作業が終ったら、push し、プルリクエストを作成してください
    - 署名付きコミットに失敗した場合は、コミットはせずコミットメッセージのみ提示してください
- `gh pr` コマンドを活用してください
    - `gh pr view` でPRの詳細を確認できます
    - `gh pr checks` でCIの状態を確認できます
    - `gh pr diff` で差分を確認できます
    - レビューコメントの確認や CI の失敗の修正に活用してください
- コードの変更は段階的に行い、各段階で動作確認をしてください
- エラーログは詳細に確認し、根本原因を特定してください
- 実装より、調査、考察、計画、仕様の更新などを優先してください

### TS/JS プロジェクト(package.json が存在するディレクトリ)の場合

- `npm run`, `pnpm run`, `bun run` などの代わりに、`nr` を使用してください。
- `bun test`, `bun build` などのbunの予約コマンドは例外としてそのまま実行してください。
