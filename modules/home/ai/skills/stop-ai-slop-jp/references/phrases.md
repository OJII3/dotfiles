## 1. 定型句と偏愛語

### 表層の定型句

語彙だけでバレるタイプ。thinkingなしの古いモデルや、未調整のAI、素のプロンプトでよく出る。

- 「いかがでしたでしょうか」「いかがでしたか」
- 「ぜひ参考にしてください」「ぜひ〜してみてください」
- 「○○の重要性を再認識しました」
- 「近年〜が注目されています」
- 「現代社会において〜は重要なテーマです」
- 「○○することが大切です」「○○することが重要です」
- 「○○に他なりません」
- 「ご質問ありがとうございます」(質問への丁寧な復唱)

### AI偏愛語

具体性を装った抽象語。チューニング済みのモデルでも残る、いまの臭いの中心。

**質感を装う語**
- 泥臭い、泥臭さ
- 手触り
- 肌感、肌感覚
- 体温、温度感
- 熱量
- 血の通った

**認知を装う語**
- 解像度(「解像度が上がる」「解像度が高い」)
- 腹落ち
- メンタルモデル
- 文脈(やたら言いたがる)

**評価を装う語**
- 本質、本質的
- 地に足のついた
- 等身大

**抽象比喩**
- 営み(「○○という営み」)
- 装置(「○○装置」)
- 設計(体験や状況に対して「設計されている」と書くクセ)
- インフラ(物理的なインフラではない用法)
- 接続(「人の接続」のような抽象用法)

**思想語**
- 思想(「○○の思想」)
- 原則(「3つの原則」)
- 哲学(「○○の哲学」)

### 必殺技造語(壮大化のための漢字熟語)

普通の感想を、必殺技みたいな漢字熟語で大きな話に仕立てるための語。`structures.md` の「体験の壮大化」と連動。1〜2個ならカッコつけで済むが、複数並ぶと一気にクサくなる。

- 真理、真実
- 結末、宿命、運命
- 境地、極致、究極
- 虚飾、装飾、虚像
- 美学、徳、品性
- 重厚感、深淵
- 禁欲的、優美、優雅、繊細
- 残酷、冷徹、冷酷、過酷
- 凝縮、結晶、結実

「枯淡の境地」「冷徹な結末」「残酷な真理」「虚飾を剥ぎ取る」「禁欲的な美学」「重厚感の欠如」のように複合的に使われると一発で AI 臭になる。普通の感想なら普通の語彙で書く。

AIが小説・エッセイ・文芸評論の文体を大量に学習している副作用。中二病的な物言いに寄せていく。

### 比喩のテンプレ語

これらの比喩を同じ記事で3回以上使ったらAI臭。

- 地図、羅針盤、コンパス
- 仕様書、設計書、レシピ
- 土台、柱、骨格
- 栄養、エンジン、潤滑油、スパイス
- DNA、車の両輪
- 触媒、起爆剤

## 2. 翻訳調

### 翻訳調動詞

主体を抽象化した分析動詞。英語の point to / suggest / emerge などの直訳。`structures.md` の「主体を消す(false agency)」の語彙的な現れなので、見つけたら「誰が」を立て直す。

- 指している(「同じ方向を指している」)
- 示している、示唆している
- 物語っている
- 浮かび上がる、見えてくる
- 対比をなしている
- 収斂する
- 整理する、考察する
- 捉える、論じる、位置づける

### アカデミック自称

ブログ本文で使うと翻訳論文っぽくなる。

- 「本稿」
- 「本記事」
- 「本論考」
- 「筆者」(地の文で)

## 3. 英語ネイティブ思考の漏れ

LLMが英語で大量に学習している副作用。日本語に直訳した結果として残るカタカナ・固有名詞のクセ。

### 横文字メタファー(IT/システム用語の比喩使用)

IT・SE用語を、本来関係ない文脈に比喩で持ち込むクセ。普通の日本語に置き換えできるところを、わざわざシステム用語で書いて先進的っぽい雰囲気を出す。

| 避ける | 使う |
|---|---|
| 思考のOSをアップデートする | 考え方を変える |
| 人生をハックする | 工夫する、コツを掴む |
| 新しい習慣をインストールする | 習慣をつける |
| マインドをリファクタリングする | 考え方を整理する |
| 頭の中にラボを持つ | 頭の中で試す |
| キャリアをポートフォリオで考える | 仕事の組み合わせで考える |
| 学びをアセットにする | 学びを積み上げる |

コンサル・スタートアップ界隈の語彙を AI が大量に学習している副作用。放っておくとこの手の比喩を撒いてくる。

### 過剰なカタカナ表記

普通の日本語に戻せるカタカナ語を、わざわざカタカナのまま書くクセ。

| 避ける | 使う |
|---|---|
| コンテキスト | 文脈、状況 |
| アライメント | 揃える、合わせる |
| コミュニケーション | 話す、伝える |
| インテグリティ | 誠実さ、一貫性 |
| コラボレーション | 共同作業、連携 |
| インプリメンテーション | 実装 |
| プライオリティ | 優先 |
| アグリー | 同意、賛成 |
| ナレッジ | 知識、ノウハウ |

カタカナを完全に排除する必要はない(技術用語は技術用語のまま)。普通の日本語で言えるところまでカタカナにしているとAI臭になる。

### 英語固有名詞の律儀さ

正式名称を毎回フルで律儀に書くクセ。

> OpenAIが発表したGPT-5は、Anthropicが発表したClaude 4と比較して、Googleが発表したGemini 3よりも...

普通の日本人はここまで正確に書かない。「ChatGPTは」「Claudeは」「Geminiも」のようにレイヤーを揃えずに混ぜたり、社名を省略したりする。フル正式名称が3つ以上並んだら警戒。

## 4. 立場を薄める語

`structures.md` の「立場から逃げる」「無害さにチューニングされている」の語彙版。AIが「間違えない」「角を立てない」ために使う逃げ。

### 結論回避フレーズ(事なかれ主義)

すべての立場を紹介してどれも選ばない、AIのデフォルト挙動。

- 「ケースバイケース」
- 「場合によります」
- 「価値観によります」
- 「状況に応じて異なります」
- 「一概には言えませんが」
- 「一般的には」
- 「メリットもあればデメリットもあります」
- 「○○という側面もありますが、一方で〜」
- 「賛否が分かれるテーマです」

### 三段ヘッジ

同じ段落で複数のヘッジを重ねるとAI臭になる。1段落に1つまで。

- 「○○ことがある」
- 「○○場合がある」
- 「○○可能性がある」
- 「○○なりうる」
- 「○○とは限らない」
- 「○○と言えなくもない」
- 「あくまで○○として捉えるのが正確だ」

### 曖昧な断言(vague declaratives)

「重要だ」「深い」「構造的だ」と言うが、具体的に何が・どう・なぜ、を書かない。

- 「本質的な問題がある」
- 「構造的な課題である」
- 「示唆に富む結果だ」
- 「重大な意味を持つ」
- 「根本的な転換が必要だ」
- 「深い洞察が得られる」
- 「重要な示唆を含む」

修正方針: 具体的な「何が」「なぜ」を書く。書けないなら、その文を削る。

### 弱い否定

はっきり「やるな」と言うべき場面で弱めるクセ。

- 「あまり推奨されません」
- 「注意が必要です」
- 「望ましくないとされています」
- 「避けたほうが良いでしょう」

修正方針: 「やるな」「ダメだ」と言い切る。

### 強い評価+根拠なし

ホメるときは具体的な根拠を添える。根拠なしの大言は逆に薄い。

- 「非常に有効です」
- 「画期的な手法です」
- 「革新的なアプローチです」
- 「大きなメリットがあります」
- 「驚くべき効果が期待できます」

## 5. 時間稼ぎと自己解説

### 咳払いの前置き(throat-clearing)

本題に入る前の時間稼ぎ。削除して直接本題に入る。

- 「実はですね、」
- 「結論から言うと」(言いつつ結論が薄い場合)
- 「正直に言うと」「率直に言いますが」
- 「ここで重要なのは」
- 「ひとつ言っておきたいのは」
- 「結局のところ」「本当のところ」
- 「あえて言うなら」

### メタコメント(自分の文章を解説するクセ)

構造を本文で予告するパターン。詳しくは `structures.md` の「メタ的な構造宣言」。

- 「以下の3つの観点から説明します」
- 「それぞれ順に見ていきましょう」
- 「ここからが本題です」
- 「補足ですが、」
- 「ステップ1」「STEP1」「Step1」
- 「○○について、改めて整理します」

## 6. 強度の振り切り

`structures.md` の「中間温度の欠如」の語彙版。全部を強い言葉で塗らない。

### 根拠なし副詞

英語の "really" "just" "literally" 系の翻訳。1段落に2個以上重なったらNG。

- 非常に、とても、かなり
- めっちゃ、ガチで、マジで
- 普通に(意味のない強調用法)
- 本当に、実に
- 確かに(意味のない同意)

### 「○○すぎる」連発

ポジティブ体験の強度を一律に振り切るクセ。1記事に2〜3回まで。

- 「美味しすぎ」「気持ちよすぎ」「優雅すぎ」「感動的すぎ」「ヤバすぎ」

全部に貼らない。中間温度(「悪くない」「まあまあ」「微妙」)を混ぜる。

### 「(笑)」の文末貼り付け

絵文字的に文末へ律儀に付けるクセ。1記事に1〜2回まで。

## 8. 機械的なアーティファクト

出力処理の痕跡として残る記号。原因は機械処理側にあるので、検索 → 全置換で機械的に潰せる。

### 強調の `**` 残骸

AIが Markdown で出力した強調記号が、消し忘れで残るパターン。

```
**SEO対策をするためには**以下のような施策が効果的です。**
```

### コロン `:` の後の半角スペース

英語ベースの処理が原因で、日本語にも `: ` のような半角スペースが残る。

```
重要な要素： テクノロジーと人間性の融合。
```

### 全角文字に紛れた半角スペース

```
SEO対策をする ためには以下の ような施策が効果的です。
```

半角スペースを検索して、不要なものを削除。

### 全角ダッシュ ──

英語の em-dash の直輸入。日本語ではコロン、改行、読点に置き換える。本文に混入していないか検索する。

### 装飾絵文字の均等撒布

🚀🎯✨💡🔥📈 のような装飾絵文字が、段落末や箇条書きに律儀に等間隔で出てくるクセ。1記事に5〜6個出たらAI臭。

真面目な文書ではそもそも絵文字を使わない。賢くないモデルほど顕著に出る。残すなら1記事に1〜2個、本当に意味のある場所だけ。
